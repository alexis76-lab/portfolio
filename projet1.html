Situation E6 : Configuration d'un Active Directory

Mise en place d'un contrôleur de domaine Windows Server avec configuration réseau, installation du rôle AD DS, promotion du serveur et vérifications de fonctionnement.

Contexte

Ce projet a été réalisé dans le cadre d'un TP de formation en BTS SIO option SISR. L'objectif était de comprendre le rôle d'Active Directory dans une infrastructure et de mettre en place un service d'annuaire exploitable.

Cette situation professionnelle est utilisée pour l'épreuve E6, car elle montre une démarche complète : préparation, installation, configuration, validation et documentation.

Environnement

Windows Server Active Directory AD DS DNS Épreuve E6
Objectif

Installer un annuaire Active Directory fonctionnel pour centraliser l'authentification et l'administration du domaine.

Mon rôle

Configurer le serveur, installer le rôle AD DS, promouvoir la machine et contrôler le bon état du service.

Résultat

Un contrôleur de domaine opérationnel, prêt à accueillir des comptes, groupes et postes clients.

Type

Projet d'école réalisé en groupe.

Durée

4 heures de travail sur la réalisation et la validation.

Période

10 octobre 2025.

Budget estimé

192,32 euros bruts en temps humain dans le cadre du TP.

Déroulement du TP

  • Analyse : identification du rôle d'Active Directory Domain Services dans le système d'information.
  • Préparation : vérification du serveur et configuration d'une adresse IP statique.
  • Installation : ajout du rôle AD DS depuis Windows Server.
  • Promotion : transformation du serveur en contrôleur de domaine.
  • Validation : vérification du domaine, du service DNS et de l'état général de l'installation.
  • Documentation : conservation du TP et de la fiche descriptive E6 comme preuves.

Difficultés et solutions

  • Adressage réseau : l'adresse IP devait être stable pour éviter les erreurs de résolution et de promotion du domaine.
  • Ordre des étapes : la configuration réseau devait être validée avant l'installation du rôle AD DS.
  • Contrôle final : les vérifications ont permis de confirmer que le serveur était bien contrôleur de domaine.
