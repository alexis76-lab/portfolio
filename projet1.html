Création d’une Active Directory sur Windows Server 2019

Mise en place d’un contrôleur de domaine fonctionnel

Contexte

Dans le cadre d’un projet d’administration système, il a été nécessaire de mettre en place une infrastructure Active Directory (AD) sous Windows Server 2019. L’objectif était de centraliser la gestion des utilisateurs, des groupes et des ressources réseau au sein d’un environnement d’entreprise simulé. Ce projet visait à renforcer mes compétences en administration Windows Server, en gestion de domaine et en configuration réseau.

Objectif

Mettre en place un contrôleur de domaine fonctionnel basé sur Active Directory afin de :

  • Centraliser l’authentification et la gestion des utilisateurs.
  • Organiser les postes clients au sein d’un même domaine.
  • Implémenter des GPO.
  • Faciliter l’administration et la maintenance du réseau.

Méthode

  • Installation de Windows Server 2019 : mise en place du système sur une machine virtuelle via Proxmox, configuration du nom d’hôte et de l’adresse IP statique.
  • Ajout du rôle AD DS : installation du rôle Active Directory Domain Services et promotion du serveur en contrôleur de domaine.
  • Création du domaine : configuration d’un domaine interne nommé btssio.local.
  • Configuration du domaine : création des UO, ajout des comptes utilisateurs et groupes de sécurité.
  • Ajout des postes clients : intégration des machines Windows au domaine et test des connexions.
  • Application des GPO : mise en place de stratégies de mot de passe et de sécurité pour les utilisateurs du domaine.

Choix techniques

  • Système d’exploitation : Windows Server 2019 Standard
  • Virtualisation : Proxmox
  • Adresse IP : statique
  • Rôle principal : Active Directory Domain Services
  • DNS : utilisé pour la résolution interne
  • Nom de domaine : btssio.local

Bilan

Le déploiement de l’Active Directory s’est déroulé avec succès. Le domaine est fonctionnel et permet la gestion centralisée des utilisateurs et ordinateurs. Les tests de connexion depuis les postes clients ont confirmé la bonne intégration au domaine.

Ce projet m’a permis de mieux comprendre la logique de fonctionnement d’un domaine Windows. Des évolutions futures pourraient inclure la mise en place d’un DNS secondaire, d’un serveur DHCP intégré et d’un second contrôleur de domaine pour la redondance.
