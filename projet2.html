Situation E6 : Installation de GLPI sur Linux

Installation d'une solution de gestion de parc et de ticketing sur Linux, avec préparation de l'environnement, base de données, configuration initiale et test d'utilisation.

Contexte

Ce projet a été réalisé dans le cadre d'un TP de formation en BTS SIO option SISR. L'objectif était d'installer GLPI, une solution de gestion de parc informatique et de support, dans un environnement Linux.

Cette situation professionnelle est utilisée pour l'épreuve E6, car elle met en avant la mise à disposition d'un service informatique et la validation d'une application web en environnement serveur.

Environnement

Linux GLPI Base de données Ticketing Épreuve E6
Objectif

Mettre à disposition une instance GLPI fonctionnelle pour gérer des tickets et découvrir la gestion de parc.

Mon rôle

Préparer l'environnement, créer la base de données, installer GLPI et valider l'accès à l'interface.

Résultat

Une instance GLPI accessible, configurée et testée avec une première utilisation.

Type

Projet d'école réalisé en groupe.

Durée

4 heures de travail sur l'installation et les tests.

Période

10 octobre 2025.

Budget estimé

96,16 euros bruts en temps humain dans le cadre du TP.

Déroulement du TP

  • Préparation : vérification de l'environnement Linux et des services nécessaires.
  • Base de données : création de la base destinée à l'application GLPI.
  • Installation : téléchargement, déploiement et configuration initiale de GLPI.
  • Paramétrage : accès à l'interface web et premiers réglages.
  • Validation : test de fonctionnement à travers l'interface GLPI.
  • Documentation : conservation du TP et de la fiche descriptive E6 comme preuves.

Difficultés et solutions

  • Dépendances serveur : l'installation demandait de préparer correctement l'environnement avant de lancer GLPI.
  • Base de données : le lien entre l'application et la base devait être cohérent pour terminer l'installation.
  • Validation : l'accès à l'interface et le test d'utilisation ont permis de confirmer que le service était opérationnel.
